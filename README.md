# Self-Consistency AI Engine with LLM Arbiter

A CLI application that queries multiple Large Language Models (LLMs) in parallel and utilizes an objective LLM judge/arbiter to evaluate, select, and explain the best candidate response. This design implements a form of **Self-Consistency** and **LLM-as-a-Judge** pattern to deliver higher quality, reliable text generations.

---

## 🏗️ Architecture & Workflow

```
               +--------------------------------------+
               |          User Prompt Input           |
               +------------------+-------------------+
                                  |
                                  v
                    +-------------+-------------+
                    |                           | (Parallel Execution)
                    v                           v
         +----------+----------+     +----------+----------+
         |     OpenAI API      |     |     Gemini API      |
         |    (gpt-4o-mini)    |     |  (gemini-2.5-flash) |
         +----------+----------+     +----------+----------+
                    |                           |
                    +-------------+-------------+
                                  |
                                  v (Candidate Responses)
                   +--------------+--------------+
                   |         OpenAI Judge        |
                   |    (gpt-4o-mini Arbiter)    |
                   +--------------+--------------+
                                  |
                                  v (Structured JSON Evaluation)
                        Winner Selected & Logged
```

1. **Parallel Execution**: When the user enters a prompt, the application initiates simultaneous requests to:
   - **OpenAI (GPT-4o-Mini)** via the new Responses API.
   - **Google Gemini (Gemini 2.5 Flash)** via the `@google/genai` SDK.
2. **LLM Evaluation**: The system feeds the user's prompt, both candidate outputs, and context history into an independent LLM judge (Arbiter) prompt.
3. **Structured Output**: The Arbiter returns a structured JSON response matching the `EvaluationJsonSchema`:
   - `winner`: The selected winning model (e.g. GPT, Gemini).
   - `reasoning`: Detailed reasoning behind the decision.
   - `winning_text`: The actual text of the chosen output.
4. **History Retention**: The winning response is appended to the chat session history. A sliding window mechanism ensures context length limits are respected.

## ⚙️ Prerequisites & Setup

Ensure you have [Node.js](https://nodejs.org/) installed (v18+ recommended).

### 1. Environment Keys
Create a `.env` file in the root folder of the project:

```env
OPENAI_API_KEY=your_openai_api_key_here
GEMINI_API_KEY=your_gemini_api_key_here
```

*Note: The application will automatically validate the environment variables on startup and guide you if keys are missing.*

### 2. Installation
Install the package dependencies:

```bash
npm install
```

---

## 🚀 Running the Application

To start the interactive command-line interface:

```bash
node index.js
```

### Features of the Interactive CLI:
* **Interactive Prompt Loop**: Engage in a continuous conversational thread with the models.
* **Animated Status Spinner**: Live terminal spinner animations showing progress during API calls.
* **Raw Candidates Preview**: Inspect side-by-side outputs generated by each individual LLM before judging.
* **ANSI Colored Layouts**: Clearly formatted, colored panels presenting the **🏆 Winner**, **🧠 Reasoning**, and **📄 Content**.
* **Automatic Chat Memory**: Session context is maintained (using a sliding window of the last 6 turns) so that follow-up prompts are answered in context.
